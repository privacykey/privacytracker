## Adding a story

Co-locate `Foo.stories.tsx` next to `Foo.tsx` in `app/components/`. Use the
canonical CSF3 shape (`Meta` default export + named `Story` exports). The
global decorators in `.storybook/preview.tsx` already wrap every story
with the i18n provider, font + globals.css, theme attribute, and the
feature-flag resolver — you only need to declare overrides via
`parameters.focus` / `parameters.flagOverrides` when the component cares.

## What's out of scope

Data-coupled views (anything that fetches `/api/*` or accepts large
server-resolved props) and server-only-tainted modules (anything that
imports `lib/db`, `lib/*-server`, or `better-sqlite3` at the top level)
don't have stories yet — they'll land in follow-up PRs once a fixtures
layer exists.
